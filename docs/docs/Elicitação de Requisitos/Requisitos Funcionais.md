# Requisitos Funcionais

Requisitos funcionais são essenciais para definir o comportamento e as capacidades de um sistema, delineando claramente o que ele deve realizar para satisfazer as necessidades e expectativas dos usuários. Estes requisitos descrevem as funções específicas que o sistema deve executar, incluindo processos, manipulação de informações, funcionalidades de interface e respostas a entradas específicas ou condições ambientais. A classificação por tipo na tabela de requisitos funcionais conecta-se diretamente a essas especificações, organizando os requisitos de acordo com suas categorias de funcionalidades ou responsabilidades dentro do sistema. Isso permite uma gestão mais eficiente e focada no desenvolvimento, garantindo que todas as áreas críticas, como interações de usuário, administração do sistema, gerenciamento de imagens, segurança e assinaturas, sejam adequadamente cobertas e priorizadas conforme suas complexidades e impactos no funcionamento global da plataforma. Assim, cada tipo de requisito não apenas define o que deve ser implementado, mas também contextualiza sua importância e papel no cumprimento dos objetivos do sistema como um todo.

| RF31  | Integridade dos diferentes níveis de acesso | Considerando a característica da aplicação de possuir diferentes tipos e níveis de assinatura, é de suma importância que os níveis de acesso para cada usuário sejam bem definidos e protegidos para garantir que imagens do cliente não possam ter sua integridade violada. | RF32  | Criptografia de informações confidenciais | É de suma importância garantir que informações confidenciais e/ou sensíveis sejam criptografadas em seu armazenamento e enquanto estiverem em trânsito, assim garantindo sua segurança mesmo sob caso de acesso indevido. | 4 | Segurança |
| RF33  | Uso de protocolos de comunicação seguros | O uso de protocolos de comunicação seguros como o HTTPS é de vital importância para qualquer aplicação por garantir a integridade e segurança do transporte de dados no ambiente web. | 4 | Segurança |
| RF34  | Restrição de acesso a informações confidenciais | Dado a natureza de múltiplas assinaturas e da existência de um administrador, é importante garantir que existam restrições de acesso às informações da aplicação, assim garantindo a integração do sistema. | 5 | Segurança |
| RF35  | Detecção e registro de alterações nos dados | O registro das alterações nos dados é uma boa prática para uma estrutura de banco de dados para garantir que as modificações possam ser monitoradas e validadas. | 3 | Segurança |
| RF36  | Recuperação de dados em caso de perda ou corrupção | É importante garantir que seja possível recuperar informações do banco de dados caso ocorra algum tipo de corrupção dos dados. Em geral, isso é feito através do registro dos dados em tabelas de backup e registros de logs. | 3 | Segurança |
| RF37  | Prevenção de ataques de injeção de SQL | Sendo o injection uma das falhas de segurança mais comuns em aplicações web segundo o OWASP, sendo assim, é de suma importância adicionar verificações tanto no frontend quanto no backend para evitar que comandos SQL e JavaScript possam ser injetados na aplicação. | 5 | Segurança |
| RF38  | Controle de Sessão | A aplicação deve implementar um controle de sessão seguro para gerenciar a autenticação contínua do usuário durante sua interação com a plataforma, garantindo que a sessão expire após um período de inatividade ou logout. | 4 | Segurança |
| RF39  | Proteção contra Brute Force | É necessário implementar mecanismos de proteção contra ataques de força bruta, como limitar o número de tentativas de login e bloquear temporariamente contas após várias tentativas fracassadas. | 2 | Segurança |
